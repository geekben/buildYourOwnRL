## 问题的起源

学完值迭代和 Q-Learning 后，一个自然的问题是：

> Q-Learning 在多个回合中反复访问同一个 (s, a) 对，每次都记录了 (s, a, r, s')。这些数据积累起来，不就能统计出 P(s'|s,a) 了吗？有了 P，不就能直接用值迭代了吗？

这个直觉完全正确。它揭示了强化学习中最核心的分野：**Model-Free vs Model-Based**。

---

## 从采样数据反推模型

### 方法：频率统计

假设在 GridWorld 中，你在状态 (0,0) 执行"向右"这个动作 100 次：
- 80 次到了 (0,1)，奖励 -0.1
- 20 次留在 (0,0)，奖励 -0.1（随机环境中滑倒了）

那你就可以估计：

```
P̂((0,1) | (0,0), →) = 80/100 = 0.80
P̂((0,0) | (0,0), →) = 20/100 = 0.20
R̂((0,0), →) = -0.1
```

对所有 (s, a) 对都做这样的统计，就得到了完整的模型估计 P̂ 和 R̂。

### 然后：直接套用值迭代

有了 P̂ 和 R̂，就可以直接用之前写的 `ValueIterationAgent`，一模一样地跑值迭代：

```
步骤 1：与环境交互，记录所有 (s, a, r, s') 转移
步骤 2：统计频率，估计模型
  P̂(s'|s,a) = count(s, a, s') / count(s, a)
  R̂(s, a)   = 所有 (s, a) 转移的平均 r
步骤 3：用 P̂ 和 R̂ 跑值迭代，得到最优策略
```

这种方法在学术上叫 **Certainty Equivalence**——把估计的模型当成真实模型来用。它是 **Model-Based RL** 的最朴素形式。

## 两大流派的对比

### Model-Free：直接学价值

```
经验 (s, a, r, s') → 直接更新 Q(s, a)
```

代表算法：Q-Learning、SARSA、DQN、PPO、SAC

**特点**：不建模，不存储 P，直接从经验中学习价值函数或策略。

### Model-Based：先学模型，再规划

```
经验 (s, a, r, s') → 建模 P̂, R̂ → 用模型规划（如值迭代）→ 得到策略
```

代表算法：Dyna-Q、MBPO、World Models、MuZero

**特点**：多了一个"建模"的中间步骤，但模型可以反复利用——建好模型后，可以跑任意多次值迭代，不需要再和环境交互。

### 详细对比

| 维度 | Model-Free (Q-Learning) | Model-Based (学模型+值迭代) |
|------|------------------------|---------------------------|
| **存什么** | Q 表：Q(s, a) | 模型：P̂(s'\|s,a) 和 R̂(s,a) |
| **学什么** | 直接学价值函数 | 先学模型，再算价值函数 |
| **存储量** | O(\|S\|×\|A\|) | O(\|S\|²×\|A\|)（转移矩阵更大） |
| **样本效率** | 低（每个经验只用一次） | 高（模型可反复利用） |
| **计算量** | 低（每步一次更新） | 高（建模 + 规划） |
| **对环境要求** | 只需沿轨迹走 | 同左（但批量采样需要可重置） |
| **鲁棒性** | 高（不依赖模型准确性） | 低（模型误差会累积） |
| **收敛后结果** | 相同的最优策略 | 相同的最优策略 |

### 信息组织方式的差异

两种方法在数学上是等价的（无限数据下收敛到同一结果），区别在于**信息的组织方式**：

```
Model-Free：
  经验 → Q(s,a)
  信息被"压缩"进 Q 值，丢失了转移细节

Model-Based：
  经验 → P̂(s'|s,a), R̂(s,a)
  信息被"展开"为完整的环境模型，保留了转移细节
```

Model-Based 保留了更多信息，所以可以更高效地利用数据。但代价是存储更大、建模更难。

## Dyna-Q：两个流派的结合

### 核心思想

Dyna-Q（Sutton, 1991）是一个优雅的混合方案：**既做 Q-Learning，又建模型，还用模型"脑补"额外经验**。

```
Dyna-Q 每一步做三件事：

① 直接学习（Model-Free，和 Q-Learning 一样）
   真实交互 (s, a, r, s') → 更新 Q(s, a)

② 模型学习（Model-Based 的建模部分）
   把 (s, a) → (r, s') 存进模型表

③ 规划（Model-Based 的规划部分）
   从模型中随机抽 k 个 (s, a)
   用模型预测 (r, s')
   用这些"虚拟经验"再更新 Q
```

### 直觉类比

```
纯 Q-Learning（Model-Free）：
  每天只在真实世界练习一次，进步慢

纯 Model-Based（学模型+值迭代）：
  先观察很多次，在脑子里建立世界模型，然后在脑子里反复练习

Dyna-Q（混合）：
  每天在真实世界练习一次（Q-Learning）
  + 晚上在脑子里回放和模拟练习 k 次（Model-Based 规划）
  → 进步最快
```

### 规划步数 k 的影响

Sutton 的经典实验表明，Dyna-Q 中规划步数 k 越大，学习越快：

```
k=0 ：纯 Q-Learning，需要 ~25 个回合学会
k=5 ：需要 ~10 个回合
k=50：需要 ~5 个回合
```

每次真实交互后多做几次"脑内模拟"，就能大幅减少所需的真实交互次数。这在**真实交互成本高昂**的场景（如机器人、自动驾驶）中极有价值。

### 批量采样 vs 逐步更新

一个相关的问题是："能不能在同一个状态尝试所有 action，一次性更新所有 Q 值？"

这个方案**可行但有隐含假设**——你需要能"回到"同一个状态：

```
批量采样方案：
  env.set_state(s)  # 重置到状态 s
  env.step(a₁)      # 尝试 action 1，记录结果
  env.set_state(s)  # 再回到状态 s
  env.step(a₂)      # 尝试 action 2，记录结果
  ...
```

在模拟环境中这没问题，但在**真实环境**中（如机器人、自动驾驶），你无法"倒带"回到同一个状态。Q-Learning 的优势恰恰是**只需要沿着一条轨迹走**，不需要回退。

而且，如果你能在每个状态尝试所有 action 并多次观察结果，你实际上就是在**通过采样重建转移模型 P̂**——这本质上就是 Model-Based RL。Dyna-Q 的 ② + ③ 步骤正是这个思路的工程化实现。

## Model-Based 的优势与风险

### 优势：样本效率

Model-Based 方法的最大优势是**样本效率高**。建好模型后，可以在"想象"中无限次练习，不需要真实交互。这在以下场景中至关重要：

- **机器人**：每次真实交互可能损坏硬件
- **自动驾驶**：真实测试有安全风险
- **医疗**：不能在真实患者上反复试错
- **金融**：每次交易都有真实成本

### 风险：模型误差累积

Model-Based 方法的最大风险是**模型不准时，规划结果可能比 Model-Free 更差**。

```
模型误差累积的例子：
  P̂ 有 5% 误差
  → 值迭代传播 1 步：误差 ~5%
  → 值迭代传播 5 步：误差 ~22%
  → 值迭代传播 10 步：误差 ~40%
  → 基于这个 V 推导的策略可能很糟糕
```

在简单的 GridWorld 中，P̂ 很容易估准（状态空间小，每个 (s,a) 都能被充分采样）。但在复杂环境中（如 Atari 游戏的像素输入），学一个准确的 P̂ 极其困难——状态空间太大，大部分 (s,a) 对从未被访问过。

### 存储成本

模型需要存储 P̂(s'|s,a)，这是一个 |S|×|A|×|S| 的三维表，比 Q 表的 |S|×|A| 大了 |S| 倍。状态空间大时，存储模型本身就不可行。

### 权衡总结

| 场景 | 推荐方法 | 原因 |
|------|---------|------|
| 真实交互便宜，环境复杂 | Model-Free | 不怕交互多，怕模型不准 |
| 真实交互昂贵，环境简单 | Model-Based | 少量交互建好模型，反复规划 |
| 两者兼顾 | Dyna-Q 等混合方法 | 真实交互 + 模型规划并行 |

## RL 的完整版图

从 Q-Learning 出发，可以看到整个 RL 领域的两大流派：

```
                      强化学习
                     /        \
            Model-Free         Model-Based
           /        \              |
      Value-Based  Policy-Based   Dyna-Q / MBPO / World Models
       /     \         |              |
  Q-Learning SARSA  REINFORCE     MuZero (AlphaGo 的后续)
      |                |
     DQN          Actor-Critic
      |              /    \
  Double DQN      PPO     SAC
  Dueling DQN
```

### 两条路线的终极形态

**Model-Free 路线**最终发展出 PPO 和 SAC——通过大量交互数据直接优化策略，不建模，但需要海量数据。

**Model-Based 路线**最终发展出 **MuZero**（DeepMind, 2020）——它学习一个隐式的世界模型（不是显式的 P̂，而是一个能预测未来的神经网络），在"想象"中用蒙特卡洛树搜索规划，达到了超人的游戏水平。MuZero 在围棋、Atari、国际象棋上都取得了顶尖表现，而且**不需要知道游戏规则**。

### 现代趋势：两者融合

当前 RL 研究的趋势是**融合两种方法**：

- **MBPO**（Model-Based Policy Optimization）：用学到的模型生成短期轨迹，用 Model-Free 方法（SAC）在这些轨迹上训练
- **Dreamer**：在学到的世界模型中"做梦"，完全在想象中训练策略
- **MuZero**：隐式模型 + 树搜索规划

这些方法的共同思路是：**用模型提高样本效率，用 Model-Free 方法保证鲁棒性**。

## 关键洞察

### 1. Q-Learning 的数据里已经包含了模型信息

Q-Learning 在训练过程中收集的所有 (s, a, r, s') 转移数据，理论上足以重建完整的环境模型。Model-Free 方法选择"丢弃"这些结构化信息，直接压缩进 Q 值；Model-Based 方法选择"保留"这些信息，构建显式模型。

这是一个**信息压缩 vs 信息保留**的权衡：
- 压缩（Model-Free）：存储小，鲁棒，但丢失了可复用的结构
- 保留（Model-Based）：存储大，脆弱，但信息可反复利用

### 2. "在脑子里练习"是 Model-Based 的核心价值

Dyna-Q 的规划步骤本质上是"在脑子里练习"——用模型生成虚拟经验，不需要真实交互。这和人类的学习方式惊人地相似：我们不仅从真实经历中学习，还会在脑海中"复盘"和"模拟"。

### 3. 模型的形式可以多样

"模型"不一定是显式的 P̂(s'|s,a) 转移矩阵。它可以是：
- **表格模型**：直接存储 (s,a) → (r,s') 的映射（Dyna-Q）
- **参数化模型**：用神经网络预测 s' = f(s, a)（World Models）
- **隐式模型**：不预测具体的 s'，而是预测"对决策有用的信息"（MuZero）

MuZero 的突破在于：它证明了**不需要准确预测下一个状态的像素**，只需要学到一个能支持规划的隐式表示就够了。

### 4. 这个分野贯穿整个 RL 发展史

从最早的动态规划（纯 Model-Based，已知模型）到 Q-Learning（纯 Model-Free），再到 Dyna-Q（混合），再到 MuZero（隐式模型），Model-Free vs Model-Based 的张力一直是推动 RL 发展的核心动力之一。
