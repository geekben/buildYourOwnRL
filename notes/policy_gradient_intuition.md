## 为什么需要策略梯度

### 策略迭代中的策略改进

在策略迭代中，策略改进非常简单：

```python
# 遍历所有动作，选 Q 值最大的
q_values = [Q(s, a) for a in [上, 下, 左, 右]]
π(s) = argmax(q_values)
```

这能工作，是因为满足两个条件：
1. **动作是离散的、有限的**（只有上下左右 4 个）
2. **可以枚举所有动作**，逐个算 Q 值，选最大的

### 连续动作空间的挑战

当动作空间变为连续时（如机械臂关节角度 ∈ [-180°, +180°]），argmax 失效：

```python
# 不可能执行！a 有无穷多个值
π(s) = argmax_a Q(s, a)
```

**连续不代表不能求 max**。理论上，如果 Q(s, a) 关于 a 是简单的可微函数（如二次函数），可以通过 ∂Q/∂a = 0 解析求极值。但现实中 Q(s, a) 通常由神经网络表示，是高维非凸函数，**解析求极值不可行**。

而且实际问题中动作往往是**多维连续空间**：
- 机械臂：6 个关节角度 → a ∈ ℝ⁶
- 自动驾驶：(油门, 刹车, 方向盘) → a ∈ ℝ³

在高维连续空间中，无论是枚举还是解析求极值都不可行，**梯度迭代逐步逼近是唯一通用的方法**。

## 对谁求梯度：动作空间 vs 参数空间

这里有一个**非常容易混淆的关键区分**。

### 方案 A：在动作空间中求极值

固定策略，对 Q 函数关于动作 a 求导：

```
∂Q(s, a)/∂a = 0 → 找最优动作 a*
```

问题：Q 函数是神经网络，非凸、高维、多极值，解不出来。

### 方案 B：在参数空间中优化策略（策略梯度）

把策略定义为带参数 θ 的函数 π_θ(a|s)（如神经网络），优化参数 θ 使期望累计奖励最大：

```
J(θ) = E[在策略 π_θ 下的累计奖励]
θ ← θ + α · ∇_θ J(θ)    （梯度上升，最大化奖励）
```

**方案 B 绕开了"对 Q 求极值"这个难题**。它不去找"哪个动作最好"，而是调整策略网络的参数，让好动作被选中的概率变大。

### 两种方案都真实存在

| 方案 | 求梯度的对象 | 代表算法 | 策略类型 |
|------|------------|---------|---------|
| A | Q 对动作 a 的梯度 | DDPG、TD3 | 确定性策略 |
| B | 目标函数 J 对参数 θ 的梯度 | REINFORCE、PPO、SAC | 随机策略 |

DDPG 通过链式法则把 Q 对 a 的梯度传回到参数 θ 上：

```
∇_θ Q(s, π_θ(s)) = ∇_a Q(s, a)|_{a=π_θ(s)} · ∇_θ π_θ(s)
                    ─────────────────────────   ──────────────
                    Q 对动作 a 的梯度            策略对参数 θ 的梯度
```

**注意**：DDPG 是**无模型**的。它的 Q 函数不是环境模型给的，而是通过与环境交互的数据学出来的（用神经网络近似）。

## 策略梯度公式的直觉理解

### 策略梯度定理

$$\nabla_\theta J(\theta) = \mathbb{E}\left[ \nabla_\theta \log \pi_\theta(a|s) \cdot Q^{\pi}(s, a) \right]$$

### 逐项拆解

**∇_θ log π_θ(a|s)** - "如何调整参数 θ，使动作 a 的概率增大"

这是策略网络关于参数 θ 的梯度方向。沿着这个方向更新 θ，动作 a 被选中的概率会增大。**它本身不知道动作好不好**，只知道"如果你想让动作 a 的概率增大，θ 应该往哪个方向调"。

**Q^π(s, a)** - "动作 a 到底有多好"

这是**策略评估**的结果，告诉你在状态 s 下执行动作 a 能拿到多少期望累计奖励。

**两者相乘** - Q 值充当权重，决定梯度方向的步长和正负：

| Q^π(s,a) 的值 | 效果 | 直觉 |
|---------------|------|------|
| Q = +10（很好） | 沿梯度方向大步更新 | 大幅增大这个好动作的概率 |
| Q = +1（一般） | 沿梯度方向小步更新 | 略微增大概率 |
| Q = -5（很差） | 沿**反方向**更新 | 减小这个差动作的概率 |

### 对应到梯度迭代的理解框架

```
经典梯度迭代：参数 ← 参数 + 步长 × 梯度方向

策略梯度：
  梯度方向 = ∇_θ log π_θ(a|s)   ← 参数空间中的方向
  步长     = α · Q^π(s, a)       ← 由动作好坏决定步长大小和正负

  θ ← θ + α · Q^π(s,a) · ∇_θ log π_θ(a|s)   ← 逐步逼近最优策略参数
```

### 关键联系：Q^π 从哪来？

策略梯度公式中的 Q^π(s, a) 正是**策略评估**的结果。评估和改进的分离结构完全保留了：

```
策略迭代：策略评估 → V^π(s)    → 用于 argmax 改进
策略梯度：策略评估 → Q^π(s,a)  → 用于加权梯度更新
```

只是"改进"的方式从 argmax 变成了梯度上升。

## 三种策略改进方式的对比

```
┌──────────────────────────────────────────────────────────────┐
│ 1. 策略迭代（表格方法）                                       │
│    策略形式：查找表 s → a                                     │
│    改进方式：argmax_a Q(s, a)，直接替换表中的映射              │
│    Q 的来源：已知模型，精确计算                                │
│    适用范围：有限状态 + 有限动作 + 已知模型                     │
├──────────────────────────────────────────────────────────────┤
│ 2. 策略梯度 / PPO / SAC（随机策略）                           │
│    策略形式：神经网络 π_θ(a|s) → 概率分布                     │
│    改进方式：∇_θ log π_θ · Q^π，按 Q 值加权调整概率           │
│    Q 的来源：无模型，从交互数据中学习                          │
│    适用范围：连续/离散动作 + 高维状态 + 未知模型                │
├──────────────────────────────────────────────────────────────┤
│ 3. DDPG / TD3（确定性策略）                                   │
│    策略形式：神经网络 π_θ(s) → 确定动作值                     │
│    改进方式：∇_θ Q(s, π_θ(s))，链式法则传梯度                 │
│    Q 的来源：无模型，从交互数据中学习                          │
│    适用范围：连续动作 + 高维状态 + 未知模型                     │
└──────────────────────────────────────────────────────────────┘
```

三种方法的共同点：都保留了策略迭代的**"评估 + 改进"框架**，区别只在于策略的表示形式和改进的具体方式。

## 确定性策略的陷阱与随机策略的回归

### DDPG 的确定性策略问题

DDPG 的策略 π_θ(s) 对每个状态只输出一个确定的动作：

```
状态 s₁ → 永远输出 a = 0.73
状态 s₂ → 永远输出 a = -0.15
```

这在本质上是一个**连续版的查找表**--中间状态通过神经网络做了平滑插值，但每个状态仍然只对应一个固定动作。

### 问题 1：探索不足 → 局部最优

确定性策略在同一状态下永远执行同一动作，**不会主动尝试其他动作**：
- 早期碰巧发现了一个"还不错"的动作，就会一直用它
- 永远不知道有没有更好的动作存在

DDPG 的解决办法是人工注入噪声（`实际动作 = π_θ(s) + 噪声`），但这种无方向的随机扰动探索效率很低。

### 问题 2：Q 函数过估计 → 过拟合

策略专门去找 Q 函数给高分的动作，但 Q 函数本身有估计误差：

```
真实 Q 值：  a=0.5 → Q=10,  a=0.7 → Q=12
估计 Q 值：  a=0.5 → Q=10,  a=0.7 → Q=18  ← 高估了！

策略选 a=0.7（因为估计 Q=18），但实际只有 12
→ 策略被 Q 函数的误差"欺骗"了
```

### SAC 的解决方案：回归随机策略

SAC 的关键洞察：**不要输出确定的动作，而是输出动作的概率分布**。

```
DDPG：π_θ(s) → a = 0.73（确定值，每次都一样）
SAC： π_θ(s) → N(μ=0.73, σ=0.1)（概率分布，每次采样略有不同）
```

SAC 还在奖励函数中加入**熵奖励**：

```
目标 = 最大化（累计奖励 + α · 策略的熵）
```

熵越大，策略越"随机"，探索越充分。从根本上解决了确定性策略的探索不足问题。

### 后续算法的改进总结

| 问题 | DDPG 的表现 | 解决方案 | 代表算法 |
|------|------------|---------|---------|
| **探索不足** | 确定性输出 + 随机噪声 | 输出概率分布，天然带探索 | SAC |
| **Q 过估计** | 单个 Q 网络，容易高估 | 双 Q 网络，取较小值 | TD3、SAC |
| **局部最优** | 确定性策略容易卡住 | 随机策略 + 熵奖励 | SAC |
| **更新不稳定** | 策略更新幅度不可控 | Clip 或 KL 约束限制步长 | PPO、TRPO |

## 演进路线总结

```
策略改进方式的演进：

argmax（策略迭代，表格方法）
  ↓ 动作空间变连续，argmax 不可行
梯度上升（REINFORCE，随机策略梯度）
  ↓ 方差太大，加入 Critic 降低方差
Actor-Critic
  ↓ 确定性策略变体
DDPG（确定性策略梯度）
  ↓ 探索不足、Q 过估计
TD3（双 Q 网络 + 延迟更新）
  ↓ 回归随机策略 + 最大熵
SAC（最大熵强化学习）
  ↓ 更新步长不好控制
PPO（Clip 限制更新幅度）/ TRPO（KL 散度约束）
```

**核心不变量**：无论算法如何演进，策略迭代的**"先评估当前策略，再据此改进"**的框架始终没变。变化的只是：
- **策略的表示**：查找表 → 神经网络
- **评估的方式**：精确计算 → 采样近似
- **改进的方式**：argmax → 梯度上升
